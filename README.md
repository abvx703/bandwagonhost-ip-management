## 换IP前要先确认问题

这是很多人省略的一步，结果花了钱问题还没解决。

IP无法访问，不一定就是被封。连不上搬瓦工VPS最常见的原因有这几类：

1. **VPS没开机** — KiwiVM里看一眼运行状态，是不是处于stopped状态
2. **SSH端口变了** — 搬瓦工VPS默认SSH端口不是22，在KiwiVM里能看到实际端口号
3. **防火墙拦截** — 服务器上的iptables或ufw规则导致连不上
4. **本地网络问题** — 换个网络环境测试（比如手机热点）
5. **IP真的被封了** — 从国内Ping全丢包，但用境外网络能正常访问

官方有检测工具，在KiwiVM控制面板找到 **Test Main IP** 功能，点击测试后出现绿色"IP NOT BLOCKED"表示IP没问题，出现红色"IP BLOCKED"才说明IP确实被封。

只有确认IP BLOCKED之后再去付费换，别冤枉$8.79。

## 换完IP之后还要做什么

很多人付完钱看到KiwiVM里IP换了就以为完事。其实后续还有几个地方要同步更新，不然服务还是挂着：

1. **SSH连接** — 重新用新IP连，端口号不变
2. **域名A记录** — 如果有绑定域名，去DNS后台把A记录从旧IP改成新IP
3. **网站面板** — 宝塔之类的面板设置里如果有写死IP，改过来
4. **防火墙白名单** — 如果有设置白名单，把新IP加进去
5. **CDN源站地址** — 用了Cloudflare或其他CDN的，改一下源站IP

这些改完，服务才算真正恢复。漏了一个，很可能网站已经能从新IP访问，但Cloudflare还往旧IP回源，结果502一片。

## 关于新IP质量：搬瓦工会保证什么

换IP后分配的是什么样的IP，很多人担心"会不会又分到一个被封的"。

官方说明是：换IP时会从可用池里随机分配一个IPv4，并验证该IP在全球各大洲均可正常访问，才会正式分配给你。也就是说，新IP会做一轮可用性检测，确认干净之后才生效。实际用下来，新IP被立刻再次封的情况相对不常见，大多数情况下换了能正常用一段时间。

讲真，如果换了新IP很快又封，通常不是IP质量的问题，而是使用方式触发了屏蔽——比如协议特征太明显、短时间高频连接等。这时候换IP治标不治本，更应该考虑换线路或者调整配置。
